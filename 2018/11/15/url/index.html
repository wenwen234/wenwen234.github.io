URL

protocol底层使用协议
slashes是否有协议的双斜线
host域名
port端口
hostname主机名
search查询字符串参数
query发给http服务器的数据
pathname访问资源路径名
herf未被解析的完整超链接，协议和主机会被小写化
开发面板：
timing:（可以知道为什么页面慢）
stalled:浏览器要发出请求到请求可以发出的等待时间
proxy negotiation:代理协商时间
request sent：请求的第一个字节发出前的时间到最后一个字节发出后的时间（请求时间/上传时间）
waiting（TTFB）：请求发出后到收到响应的第一个字节的时间
content download:收到响应的第一个字节开始到接受完最后一个字节的时间（下载时间）
headers:
remote address（远端服务器IP地址）：
request URL：请求地址
request method：请求方法
request headers：请求头
status code：响应状态码

请求头（点击view parsed查看原始）：
例如：GET /course/list HTTP/1.1 请求行：请求方法 请求路径 协议版本

get：获取数据
post：向指定的资源提交数据
put：向指定的资源位置上传一个最新内容（更新）
delete：删除
head：发出请求

状态码：
第一个数字定义类别，有5种
1×× 请求接受，继续处理
2×× 请求接受，已经处理了
3×× 重新下，有进一步操作
4×× 有错误，服务端无法实现
5×× 服务器端错误
常用：
200 客户端请求成功
400 客户端请求语法错误
401 请求未经授权
403 客户端拒绝提供服务，如没权限
404 没找到，请求资源不存在
500 服务器端发生不可预期的错误
503 服务器端不能处理当前请求，过段时间恢复正常

进阶
回调：将后续逻辑存放在回调函数中作为起始函数的参数，逐层嵌套
同步，异步
